# Blog 2: Tool 사용기 - 지식관리 도구 대탐험

## 도구 선정 기준
- 체험할 도구들을 선정할 때 다음 기준들을 고려했습니다
> - **접근성** : 모바일과 데스크톱 모두에서 사용 가능
> - **연결성** : 정보 간의 연관관계를 표현할 수 있는지
> - **지속성** : 장기간 사용했을 때도 유용한지
> - **학습 곡선** : 사용법을 익히는 데 드는 시간과 노력

**결론**: 즉석 메모와 간단한 아이디어 기록용으로는 최고. 하지만 체계적인 지식관리에는 한계가 명확함.

## 2단계: 올인원 솔루션 도전

### Notion

**사용 기간**: 4개월 집중 사용

**장점**:
- 블록 기반의 유연한 구조
- 데이터베이스 기능으로 체계적 정리 가능
- 템플릿이 풍부함
- 팀 협업 기능 우수
- 웹 클리퍼로 외부 콘텐츠 수집 용이

**단점**:
- 로딩 속도가 느림 (특히 모바일)
- 오프라인 사용 제한적
- 복잡한 구조를 만들수록 유지보수 어려움
- 가격이 상대적으로 비쌈

**구축한 시스템**:
```
🗂️ Dashboard
   ├── 📊 일일 체크리스트
   ├── 📚 학습 진행 상황
   └── 🎯 목표 트래킹

🗂️ Knowledge Base
   ├── 💻 개발
   ├── 📈 투자
   ├── 📖 독서노트
   └── 🎬 영상 정리

🗂️ Archive
   ├── 완료된 프로젝트
   └── 참고자료
```

**실제 사용 경험**:
- 초기 2주: 시스템 구축에 몰입, 매우 만족
- 1개월 후: 복잡해진 구조 때문에 접근성 저하
- 3개월 후: 단순한 메모도 Notion에서 하게 되어 속도 저하
- 4개월 후: 결국 간단한 메모는 다른 도구 사용

**결론**: 프로젝트 관리와 체계적인 문서화에는 탁월하지만, 일상적인 지식관리에는 과할 수 있음.

### Obsidian

**사용 기간**: 5개월 집중 사용 (현재까지 가장 오래 사용)

**장점**:
- 마크다운 기반으로 빠른 작성
- 링크와 백링크로 지식 간 연결 우수
- 로컬 저장으로 빠른 속도와 데이터 소유권
- 플러그인으로 무한 확장 가능
- 그래프 뷰로 지식 연결 시각화

**단점**:
- 초기 설정이 복잡함
- 모바일 앱의 기능 제한
- 동기화를 위해서는 별도 비용 필요
- 학습 곡선이 가파름

**활용 중인 플러그인**:
```
📦 Calendar - 일일 노트 관리
📦 Templater - 템플릿 자동화
📦 DataView - 데이터 쿼리
📦 TagWrangler - 태그 관리
📦 Advanced Tables - 테이블 편집
```

**폴더 구조**:
```
📂 1. Project (진행 중인 프로젝트)
📂 2. Area (관심 영역별 정리)
📂 3. Resource (참고자료)
📂 4. Archive (완료된 내용)
📂 99. Settings (템플릿, 설정)
```

**결론**: 지식 연결과 체계적 정리에 가장 적합. 다소 복잡하지만 투자한 시간만큼 가치를 제공함.

## 3단계: AI 도구의 등장

### ChatGPT + Custom GPTs

**사용 기간**: 6개월 (현재 진행형)

**활용 방식**:
- 정보 요약 및 정리
- 아이디어 브레인스토밍
- 학습 내용 질의응답
- 글쓰기 구조화 도움

**구축한 Custom GPT**:
```
📝 Knowledge Organizer
- 정보를 PARA 방식으로 분류
- 핵심 내용 추출 및 요약

🔍 Research Assistant  
- 주제별 심화 조사
- 다양한 관점 제시

✍️ Writing Coach
- 글 구조 개선 제안
- 논리적 흐름 점검
```

**결론**: 정보 처리와 아이디어 발전에 혁신적 도움. 하지만 최종 정리와 보관은 여전히 다른 도구 필요.

### Perplexity AI

**사용 기간**: 3개월

**활용법**: 실시간 정보 검색과 출처 확인이 필요한 리서치

**장점**: 검색과 요약이 동시에 이루어짐
**단점**: 개인 지식베이스와의 연결 어려움

- **2순위**: Google Keep (시각적 정리 필요시)

### 심화 학습 & 지식 정리
- **1순위**: Obsidian (연결성과 검색성 중시)
- **2순위**: Notion (프로젝트와 연계 필요시)

### 정보 처리 & 아이디어 발전
- **1순위**: ChatGPT (빠른 분석과 구조화)
- **2순위**: Perplexity (실시간 리서치)

### 프로젝트 관리 & 협업
- **1순위**: Notion (팀 협업 중시)
- **2순위**: Obsidian (개인 프로젝트)

## 도구 선택의 핵심 인사이트

### 1. 완벽한 도구는 없다

6개월간의 실험을 통해 깨달은 가장 중요한 사실은 **모든 요구사항을 만족하는 완벽한 도구는 존재하지 않는다**는 것입니다.

### 2. 워크플로우가 도구를 결정한다

도구를 먼저 정하고 사용법을 맞추려 하지 말고, **내 워크플로우에 맞는 도구를 선택**하는 것이 중요합니다.

### 3. 도구의 조합이 답이다

하나의 도구로 모든 것을 해결하려 하지 말고, **각 단계별로 최적의 도구를 조합**하여 사용하는 것이 효과적입니다.

### 4. 간단함이 지속가능성을 만든다

복잡한 시스템은 초기에는 인상적이지만, **장기적으로는 간단한 시스템이 더 지속가능**합니다.

## 현재의 통합 워크플로우

이 모든 실험을 바탕으로 현재 구축한 워크플로우입니다:

```
📱 캡처 단계
   └── Apple 메모 / Google Keep (즉석 아이디어, 링크)

🧠 처리 단계  
   └── ChatGPT (정보 분석, 구조화, 요약)

📚 정리 단계
   └── Obsidian (체계적 보관, 연결, 검색)

🎯 활용 단계
   └── Notion (프로젝트 관리, 협업)
```

## 마치며

도구 탐험을 통해 깨달은 것은 **도구는 수단이지 목적이 아니다**라는 점입니다. 화려한 기능보다는 내 루틴과 사고방식에 맞는 도구를 찾는 것이 더 중요했습니다.
